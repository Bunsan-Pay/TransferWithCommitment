コントラクトアドレスと対応チェーン ID。

デプロイ先アドレスは通常ソース上の定数です（ランタイムの環境変数差し替え API はありません）。

integration-test フィーチャー有効時のみ、transfer_with_commitment_address が環境変数 ETH_TWC_CONTRACT_ADDRESS を参照します（Anvil 結合テスト用）。

§本番・通常ビルド（管理者向け）

通常ビルドでは実アドレスは 非公開の COMPILE_TIME_TWC で決まります。transfer_with_commitment_address は integration-test 以外ではそれをそのまま返し、TRANSFER_WITH_COMMITMENT_ADDRESS は同じ値のコンパイル時定数です（関数が定数を読むわけではなく、いずれも同一ソースから派生します）。

既定（test-config なし）ではゼロアドレスのままです。運用では config.rs 内の COMPILE_TIME_TWC の not(test-config) ブランチを、デプロイ済みコントラクトの address!("0x…") に差し替えてからビルドしてください。
