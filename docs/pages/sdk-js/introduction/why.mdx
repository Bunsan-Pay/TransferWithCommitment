# Why（目的と境界）

## この SDK がする 3 つのこと

1. **viem と同じ開発体験**で、`TransferWithCommitment` に型付きアクセスする。
2. **EIP-712 署名**を、**オンチェーンの `eip712Domain()`** と矛盾しない形で取得する（ゼロ `salt` の除去など）。
3. **オフチェーン検証**の入口として、RPC レシートから `TransferWithCommitmentSent` を取り、**期待パラメータと突き合わせる**。

## 意図的にやらないこと

| やらないこと | 理由 |
| --- | --- |
| `commitment` の生成ヘルパ | アプリのスキームとハッシュ選択を固定したくない |
| ランタイムの環境変数だけで TWC アドレス変更 | 公開 API はない。アプリは **インストールしたパッケージに焼き込まれた定数**を使う。埋め込み作業は [Contributing: JS リリース](/contributing/sdk-js-release)（コントリビュータ向け）または fork／パッチ |
| チェーン allowlist | **持たない**。任意チェーンで canonical TWC がデプロイされていれば利用可。未デプロイは `eth_getCode` で検出 |
| ルートからの一括 import | バンドル境界と依存を明確にする |
